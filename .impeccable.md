# Biometrics House — Impeccable Design Context

This file is the source of truth for design decisions on this project. All Impeccable skills (`/polish`, `/critique`, `/layout`, `/typeset`, etc.) read it before doing any work. Keep it up to date when the direction shifts.

## Design Context

### Users

**Primary audience:** Decision-makers at companies sourcing an event venue in Bratislava — marketing managers, executive assistants, event agencies, HR leads planning offsites. They are comparing 3-5 venues in parallel and want to make a shortlist in under 10 minutes.

**Context of use:**
- **Desktop, during work hours** — primary. They're at their office, likely with 8+ tabs open, evaluating venues side-by-side. Attention is fractured; the site must earn attention in the first 2 seconds.
- **Mobile** — secondary but non-negotiable. Used for quick "show this to my boss" moments and evening browsing. Must feel native, not shrunk-down desktop.

**Job to be done:** "I need to find a venue that will make my event look impressive to my stakeholders, with proof it's not a generic conference hotel, and a clear path to booking."

**Emotional goal on arrival:** curiosity, then confidence. The visitor should feel "this isn't like the other venue sites I just looked at" within 3 seconds, and "these people are serious — I trust them with my event" within 30 seconds.

### Brand Personality

**Three words:** futuristic · exclusive · technically precise.

**Voice:** confident but not shouting. More museum placard than sales pitch. Short declarative sentences. Slovak primary, English for labels/numbers where it feels natural (the brand already does this — "BIOMETRICS EVENTS", "07 — FOR YOU" section numbering). Numbers and specs carry more weight than adjectives ("847 senzorov" is stronger than "smart building").

**Tone feels like:** an exhibition catalog for a building that happens to be for rent, not a brochure.

### Aesthetic Direction

**Theme: LIGHT mode, primary.**

This is the hardest and most differentiating decision on the project. Every other Bratislava venue site is dark-navy-and-gold "luxury corporate" template. Going light is the anti-template move and directly supports "exclusive" — in 2026, restraint reads as expensive, abundance reads as cheap. Light also suits the daytime desktop browsing context.

The brand's deep blues (`#031C40`, `#1b00e2`, `#0057ff`) move from being the *dominant surface* to being **precise accents**: section numbers, key words, CTA buttons, data labels, chart strokes. The dominant surface becomes an off-white tinted slightly toward the brand hue — not pure `#fff`.

Dark sections are permitted but earned, not default. Reserve them for:
- Hero (full-bleed interior photography with glass overlay — keeps the existing dramatic entrance)
- Building explode sequence (the 280-frame scroll sequence reads better against dark)
- One "vision" / manifesto moment per page

Everything else (venue grid, pillars, audience, booking flow, content) goes light.

### Reference Anchors (what "good" looks like here)

- **Shopify Editions Winter 2026** — editorial scroll storytelling, huge display type, generous whitespace, restrained color used as punctuation. Copy this approach to hierarchy and rhythm.
- **Terminal Industries** — industrial precision, monospace-ish data labels, technical-schematic feel, photography carries the narrative. Copy this approach to specs, numbers, and how labels look.
- **Mont-Fort** — luxury boutique restraint, refined typography, confident photo use, asymmetric editorial layouts. Copy this approach to "exclusivity feels like empty space, not gold leaf."
- **Anime.js** — motion as content, not decoration. Copy this approach to the scroll-driven sequences already in progress (building explode, hero zoom) — motion must *mean* something, not just happen.

**Common thread across all four:** bold typographic hierarchy, asymmetric layouts, photo/motion as primary content, color used as punctuation not as wallpaper. None of them use glassmorphism as a default. None of them use dark mode as a default.

### Anti-References (what to avoid)

- Generic Bratislava venue sites (dark navy, gold accents, stock photos of clinking champagne glasses, "premium experience" headlines)
- Crypto/SaaS landing pages (gradient text, neon on dark, glow borders, cyan `#22D3EE` accents — **remove this color from the palette**, it's the AI dark-mode tell)
- Corporate real-estate sites (identical card grids, stat counters, iconography above every heading)
- Glassmorphism as default decoration (reserved for ONE place: hero overlay on top of interior photography, where it has a reason to exist)

### Typography

**Brand font: Montserrat — fixed by brand manual.** Do NOT propose replacements. Typography skills (`/typeset`, `/polish`) may refine type scale, weight strategy, tracking, line-height, tabular numerals, and hierarchy — but the family stays Montserrat.

**Direction:**
- Use weights 400 (body), 500 (emphasis), 600 (labels/UI), 700 (headings), 900 (reserved for hero display moments).
- Build a clear modular scale with at least 1.25 ratio between steps — don't rely on size alone, combine size + weight + color + space.
- Numerals must be tabular where they carry data (capacity "300 ľudí", "847 senzorov"). Non-negotiable for "technically precise."
- Tight tracking (-0.02em to -0.035em) on large display sizes; default tracking on body; slightly open on small caps labels.

### Color — Revised Palette (light mode)

Convert existing tokens to OKLCH, drop cyan, tint neutrals toward brand blue:

```
--paper:       oklch(98% 0.005 260)    /* near-white, tinted toward brand blue */
--paper-deep:  oklch(95% 0.008 260)    /* alternate section background */
--ink:         oklch(18% 0.04 260)     /* primary text — NOT pure black, tinted */
--ink-muted:   oklch(45% 0.02 260)     /* secondary text */
--ink-faint:   oklch(70% 0.015 260)    /* tertiary / metadata */
--rule:        oklch(90% 0.01 260)     /* borders, dividers */

--brand:       oklch(32% 0.24 265)     /* #1b00e2 in OKLCH — primary accent */
--brand-azure: oklch(52% 0.22 260)     /* #0057ff — secondary accent */
--brand-sky:   oklch(68% 0.17 255)     /* #4590ff — tertiary / data viz */
--brand-deep:  oklch(22% 0.12 255)     /* #031c40 — dark sections only */

/* retire: #22D3EE (cyan), #FFFF00 (yellow) — unless re-introduced with specific purpose */
```

Yellow `#FFFF00` currently exists in tokens but isn't used meaningfully. Either find it one high-stakes role (e.g., booking confirmation, "live now" indicator) or remove it from the palette.

### Layout & Space

- 4pt base scale with semantic tokens (`--space-xs/sm/md/lg/xl/2xl`), not pixel-named.
- Break the current uniform padding (`64px 60px` everywhere). Vary vertical rhythm section-to-section — some sections should breathe much more than others. The current `bh-pillars` and `bh-audience` sections with identical 120px padding and 2/3-column card grids are the single biggest anti-pattern to fix.
- Kill the "identical card grid" pattern in `bh-pillars` (4 identical boxes) and `bh-audience` (3 identical boxes). Replace with editorial asymmetric layouts — e.g., one large hero card + three smaller, or a vertical-rhythm list with oversized numerals, or a magazine-style multi-column.
- Line length cap: 65-75ch for body copy. Currently unbounded in multiple places.

### Motion

The scroll-driven sequences already in place (hero zoom, building explode) are on-brand for the Anime.js reference — keep investing there. New rules:
- Use `transform` and `opacity` only (current inline styles are mostly fine).
- Easing: exponential ease-out (`cubic-bezier(0.16, 1, 0.3, 1)` — already in use in `brand.css`, keep it).
- No bounce, no elastic, no spring overshoot.
- Respect `prefers-reduced-motion` — currently not implemented, must be added when `/optimize` runs.

### Accessibility Baseline

- WCAG AA minimum on all text/background combinations. Current dark mode with 35% opacity white (`--text-muted: #FFFFFF59`) almost certainly fails — the light-mode migration is a chance to fix this properly.
- Focus states on every interactive element. Current `.btn` styles have no visible focus ring.
- Slovak screen reader support: `lang="sk"` is set (good). Preserve on all future sections.

## Design Principles (applied by every Impeccable skill)

These are the 5 rules every subsequent design decision must answer to. When a choice conflicts with one of these, the principle wins.

1. **Restraint is the brand.** Empty space is the luxury signal. If a section feels "too simple," it's probably right. If it feels "full," cut something.
2. **Numbers over adjectives.** "300 ľudí", "847 senzorov", "40% energy offset" carry more weight than "premium", "exclusive", "state-of-the-art". Every section should earn its place with one specific fact.
3. **Editorial, not corporate.** Asymmetric layouts, oversized numerals, confident photography. No identical card grids. No icon-above-heading template. No stat counter rows.
4. **Light surface, dark punctuation.** Light mode dominant. Deep blues as accents — section numbers, CTAs, key words, data marks. Dark sections earned, not default.
5. **Motion means something.** Every animation must convey state, hierarchy, or narrative. Decorative motion is banned. The building explode and hero zoom are content; aspire to that bar for every new interaction.

## Workflow Notes

- **Pencil is the source of truth for tokens.** When variables drift between `css/variables.css` and the `.pen` file, the Pencil file wins. Re-sync via MCP `batch_get` before making token changes.
- **Typography, palette, and layout pivots are big moves** — route them through `/shape` or `/critique` first, don't apply them blindly with `/polish`.
- **Start any session with `/critique` on the target section** to get a read on what's working before changing anything.
